*New here? [Start with the whole system](/blog/start-here-the-whole-system) — what EarlBear is and the stack this runs on. This post is one profile in [the agent fleet](/blog/the-agent-fleet).*

Watson is the default agent in the fleet — the one that runs when no other `AGENT`
is named. Its job is narrow and it runs overnight: work the EARL Jira board, draft
what each issue needs, and leave every draft for a human to accept in the morning.
It is the fleet's detective — it does the casework and writes it up, but it does not
close the case.

## What Watson picks up

Watson does not scan the whole board. Every run it pulls a scoped queue: issues
**assigned to `watson@earlbear.com`**, plus issues **labeled `ai-eligible`**. That
label is the opt-in — a human decides an issue is safe for an agent to touch, and
Watson only ever works the set it was handed. Anything unlabeled and unassigned is
invisible to it.

The board itself is a company-managed Kanban with a fixed set of columns, and the
column an issue sits in is the whole instruction. Watson is **column-aware**: it does
not guess what an issue needs from the text, it reads the column and routes to the
skill that column calls for.

## The loop, column by column

EARL runs the standard progression — Prioritized, Planning, In Progress, Review,
Done — and each column maps to a runtime skill. Watson walks its queue, and for each
issue does the one thing the column asks for.

The skills behind each column are small and named for what they do — `pull-work`
to fetch the queue, `plan-issue`, `draft-artifact`, `review-work`, and `check-out`
to close the run. The routing is the point: an issue's column is a contract that
says which of those runs.

## Why it never marks work done

The one rule that shapes everything else: **Watson never self-approves and never
moves an issue to Done.** It advances issues forward through the board, but the last
column is a wall it does not cross. Done is a human decision, every time.

That is not a limitation to be lifted later — it is the design. Watson works
overnight precisely so that a person can arrive in the morning to a board of drafts,
not a board of changes that already shipped. The agent does the labor; the human
keeps the judgment. Everything Watson produces is a proposal waiting for a yes.

## What you find in the morning

After a run, the work shows up in three places. On the **Jira board**, the issues
Watson touched have moved forward a column and carry its drafts as attachments and
its notes as comments. In **Google Drive**, the longer artifacts it produced sit as
files. And in **Supabase**, an `agent_runs` row (and per-issue `agent_issue_runs`
rows) records that the run happened and what it touched — the telemetry that lets us
see, later, what the fleet did overnight without reading every transcript.

So the morning ritual is review, not archaeology: open the board, read the drafts,
accept the ones that hold up, and move those to Done yourself. Watson did the night
shift; the sign-off is still yours.
